Microsoft Visual SourceSafe 2005 Readme

 

1. System Requirements

 

System Requirements for Installing Visual Studio 2005
Processor
  • 600 megahertz (MHz) Pentium processor
Operating System

Visual SourceSafe 2005 can be installed on any of the following systems:

  • Microsoft Windows 2003 Server, Service Pack 1.
  • Windows XP, Service Pack 2.
  • Windows 2000, Service Pack 4.
RAM
  • 192 megabytes (MB)
Hard Disk
  • 200 MB of available space required on installation drive
  • 200 MB of available space required on system drive
CD or DVD Drive Not required
Display

Minimum:

  • 800 x 600 256 colors

Recommended:

  • 1024 x 768 High Color - 16-bit
Mouse Microsoft mouse or compatible pointing device

 

Notes

  • When you start the Visual Studio 2005 installer, the default installation location is your system drive, which is the drive that boots your system. However, you can install the application on any drive. Regardless of the application's location, the installation process installs some files on your system drive. Consequently, ensure that the above amount of space is available on your system drive regardless of the application's location, and ensure that additional space, as listed above, is available on the drive on which you install the application.

 

2. Uninstalling Previous Versions of Visual SourceSafe

If you have installed any previous versions of Visual SourceSafe, such as Visual SourceSafe 6.0d or any other previous version, then you must uninstall it before beginning to install Visual SourceSafe 2005.

SourceSafe 2005 should not be installed side-by-side with any previous version of SourceSafe, otherwise the correct operation of either version can not be guaranteed. Furthermore, there is no need to keep previous versions of the product installed, because SourceSafe 2005 can operate against existing SourceSafe 6.0 databases.

 

3. Uninstalling Previous Versions of Visual Studio 2005

If you have installed previous versions of Visual Studio 2005, such as Beta 2 or Community Technical Preview (CTP) builds of Visual Studio Team Suite, Visual Studio Standard or Visual Studio Professional, then you must uninstall in the exact order below before beginning to install any version of Visual Studio 2005.

  1. Go to the Control Panel and launch Add/Remove Programs
  2. Remove "Microsoft Visual Studio 2005 Professional" or other related IDE installs such as (Visual Studio Professional/Standard/Enterprise Architect/Team Suite, etc.)
  3. Remove "Microsoft SQL Server 2005 Express Edition"
  4. Remove "Microsoft SQL Server 2005 Tools Express Edition"
  5. Remove "Microsoft SQL Native Client"
  6. Remove "Microsoft Visual Studio 64bit Prerequisites Beta"
  7. Remove "Microsoft MSDN Express Library 2005 Beta"
  8. Remove "Microsoft Visual Studio Tools for Office System 2005 Runtime Beta"
  9. Remove "Microsoft Device Emulator 1.0 Beta"
  10. Remove all Microsoft Visual Studio Versions (such as Visual Studio 2005 Professional Beta)
  11. Remove "Microsoft .NET Compact Framework 2.0 Beta"
  12. Remove "Microsoft SQL Mobile 2005 Development Tools"
  13. Remove "Microsoft Visual J# .NET Redistributable 2.0 Beta Language Pack"  (This step is not needed if you have only the English Edition)
  14. Remove "Microsoft Visual J# Redistributable Package 2.0 Beta".  If you receive an error message, see Note 1.
  15. Remove "Microsoft .NET Framework 2.0 Beta Language pack"  (This step is not needed if you only have the English Edition.)
  16. Remove "Microsoft .NET Framework 2.0 Beta".  If you receive an error message, see Note 2
  17. Remove "Microsoft Visual SourceSafe 2005 Beta"

4. Installing Visual SourceSafe 2005

4.1. Installing on Windows 2000, Windows XP, and Windows Server 2003

  • Insert the CD labeled Visual SourceSafe 2005
    Autorun starts Setup.exe. If Autorun is disabled, run Setup.exe from the root of the CD using Windows Explorer.
    Note   Visual SourceSafe 2005 cannot be installed to a mapped drive. You must install Visual SourceSafe on a local hard drive. Do not attempt to use the "map network drive" functionality inside the setup program.

4.2. Uninstalling Visual SourceSafe 2005

To uninstall Visual SourceSafe 2005

  • Use Add/Remove from the Control Panel.
    Note   To reinstall Visual SourceSafe, make sure that you have uninstalled using the above step. Then return to step 4.1 above to reinstall.

 

5. Known Issues

5.1   The processor on this machine is not supported. 

Installation of Visual Studio 2005 on the Intel Itanium (IA64) is not supported.

To resolve this issue

There is no workaround for this issue.

 

5.2   Installation Path Limitations. 

You cannot install Visual SourceSafe in a directory that uses extended characters, such as double byte characters or any character over 127 in the ANSI table.

To resolve this issue

Install to a path that only contains characters under 127 in the ANSI table.

 

5.3   SourceSafe 2005 Web Service and IIS. 

If you are installing SourceSafe 2005 on a server computer where you plan to enable the SourceSafe Web Service, you should make sure that IIS is installed before configuring the SourceSafe Web Service.

To resolve this issue

Once SourceSafe 2005 is installed, follow the instructions on the topic titled "How to: Enable the Internet Service for Remote Access".

 

5.4   SourceSafe 2005 Web Service and Windows XP. 

If you are installing SourceSafe 2005 on a computer where you plan to enable the SourceSafe Web Service, we recommend you install on a Windows Server version. Windows XP only supports a maximum of 10 concurrent client incoming sessions, which is sub-optimal if you plan to share your SourceSafe Web Service with other team members.

To resolve this issue

Once SourceSafe 2005 is installed, follow the instructions on the topic titled "How to: Enable the Internet Service for Remote Access".
